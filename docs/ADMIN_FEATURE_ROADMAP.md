# Admin 后台功能新增与优化大纲（红蓝对抗项目）

> 目标：在现有“场次 + 队伍 + 指令推送 + 大屏联动”的基础上，补齐赛事运营、裁判治理、复盘分析、系统安全四类能力。

## 1. 赛事运营（建议优先级：P0）

### 1.1 场次模板系统
- 新增“场次模板”：预置城市、战术类型、音频配置、积分规则。
- 创建场次时支持“一键套用模板”，减少人工重复配置。
- 支持模板版本号与变更记录。

### 1.2 回合制与赛程编排
- 新增赛程模块：分组赛/淘汰赛/积分赛。
- 支持自动生成轮次与对阵表。
- 支持回合计时器（开始、暂停、延时、结束）并同步到大屏。

### 1.3 任务工单化
- 把攻击事件、裁判加扣分、广播通知统一成“任务流”。
- 支持待处理、处理中、已完成状态。
- 支持负责人指派，避免多人操作冲突。

## 2. 裁判治理（建议优先级：P0）

### 2.1 审批流（双人复核）
- 关键动作（手动加分/扣分、全库重置）增加二次确认与复核人。
- 记录审批链路：发起人、复核人、时间、理由。

### 2.2 证据附件与判罚依据
- 每次判罚支持上传截图/日志/报告附件。
- 统一“判罚原因标签”（越权、注入、误报等）。
- 支持导出“判罚证据包”。

### 2.3 争议处理机制
- 增加“申诉”入口与时限规则。
- 争议处理过程可追踪（提交、复核、结论）。

## 3. 复盘分析（建议优先级：P1）

### 3.1 事件时间线增强
- 当前事件流增加过滤器：按队伍、攻击类型、状态、分值范围筛选。
- 支持关键事件打标（高风险、逆转点、争议点）。
- 支持快照书签，一键跳转回放起点。

### 3.2 数据洞察看板
- 新增 KPI 卡片：平均响应时延、有效攻击率、溯源成功率、净得分趋势。
- 新增趋势图：回合维度、战术维度、队伍维度。
- 支持按场次对比（本场 vs 历史均值）。

### 3.3 复盘报告自动化
- 一键生成赛后报告（PDF/Markdown）。
- 自动汇总：事件摘要、判罚记录、TOP 战术、关键时间点。
- 支持“导出给领导版（简版）/技术版（详版）”。

## 4. 系统安全与稳定性（建议优先级：P0）

### 4.1 RBAC 权限模型
- 角色拆分：超级管理员、裁判长、裁判员、观察员。
- 功能级权限控制（如“只读大屏控制，不可改分”）。

### 4.2 操作审计与追踪
- 所有后台动作落审计日志：谁、何时、做了什么、参数变更前后。
- 支持按时间/操作人/模块检索。

### 4.3 风险操作防护
- 高危操作增加动态口令或二次认证。
- 提供“演练模式”和“正式模式”，避免误操作影响生产。

## 5. 体验优化（建议优先级：P1）

### 5.1 批量操作
- 批量导入队伍（CSV/Excel）。
- 批量编辑成员、初始分、战术白名单。

### 5.2 快捷指令
- 常用广播模板、常用裁判判词模板。
- 键盘快捷键（发布广播、开始回放、暂停音乐）。

### 5.3 异常提示与恢复
- API 失败重试机制、网络断连提示。
- WebSocket 断线自动重连及可视化状态灯。

## 6. 迭代建议（落地节奏）

### 里程碑 M1（2 周）
- 场次模板、RBAC 基础版、审计日志基础版、事件筛选器。

### 里程碑 M2（2-3 周）
- 审批流、证据附件、赛程编排、复盘 KPI。

### 里程碑 M3（2 周）
- 自动复盘报告、争议申诉流、批量导入导出。

---

## 当前代码基础上的最小切入点
1. 先做“事件筛选 + 审计日志”两个功能，风险低、收益高。
2. 再做“场次模板”，可显著降低赛事准备成本。
3. 最后补“审批流 + 证据附件”，完善治理闭环。
